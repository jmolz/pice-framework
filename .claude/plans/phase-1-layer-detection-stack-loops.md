# Feature: PRDv2 Phase 1 — Layer Detection + Stack Loops Core

> Generated by `/plan-feature` — read CLAUDE.md, PRDv2.md, `.claude/rules/stack-loops.md`, and `.claude/rules/daemon.md` before implementing.

## Overview

Implement per-layer PICE evaluation: detect a project's architectural layers via a six-level heuristic stack, parse `.pice/layers.toml` configuration, and run sequential per-layer evaluation loops with context-isolated evaluators. This is the foundation for all v0.2 user-visible features — seam verification, adaptive evaluation, worktree parallelism, and review gates all depend on layers existing.

**Design spec:** `docs/superpowers/specs/2026-04-13-phase1-layer-detection-stack-loops-design.md`

## User Story

As a developer running `pice evaluate`, I want PICE to evaluate each layer (backend, database, API, frontend, infrastructure, deployment, observability) independently with its own contract, so that cross-layer blind spots — missing env vars, schema drift, broken deploys — are caught before I ship.

## Success Criteria

- [ ] Six-level layer detection produces non-empty `layers.toml` for 6 framework fixtures (Next.js, FastAPI, Rails, Express, SvelteKit, Nx monorepo)
- [ ] `.pice/layers.toml` parser validates schema, rejects cycles, handles all fields from PRDv2 lines 778–827
- [ ] `pice layers detect` prints proposed layers.toml; `--write` writes it; `--force` overwrites
- [ ] `pice layers list` renders table from current `layers.toml`
- [ ] `pice layers check` reports unlayered files with suggestions
- [ ] `pice layers graph` renders ASCII DAG; rejects cycles with error
- [ ] Stack Loops orchestrator runs sequential per-layer evaluation when `layers.toml` exists
- [ ] Without `layers.toml`, evaluation degrades to v0.1 single-loop behavior with a warning
- [ ] Context isolation: each layer's evaluator sees ONLY its own contract + filtered diff + CLAUDE.md
- [ ] File-level multi-layer tagging: `pages/api/users.ts` belongs to api + frontend layers via glob overlap
- [ ] Monorepo detection identifies multiple stacks for Nx/Turborepo fixtures

- [ ] Provider protocol extends `session/create` with optional `layer`, `layerPaths`, `contractPath` fields (backwards compatible)
- [ ] Per-layer results recorded in verification manifest at `~/.pice/state/{feature-id}.manifest.json`
- [ ] Override file (`.pice/layers.toml`) always wins detection — when present, detection is skipped entirely
- [ ] `always_run` layers (infrastructure, deployment, observability) evaluate regardless of change scope
- [ ] Dependency cascade is direct only (not transitive)
- [ ] All existing 271 Rust + 49 TS tests still pass (zero regressions)
- [ ] All validation commands pass with zero errors

### Task 4: Create detection engine — `pice-core::layers::detect`

**File:** `crates/pice-core/src/layers/detect.rs`
**Action:** CREATE
**Description:** Implement the six-level heuristic detection stack. Pure function: `pub fn detect_layers(project_root: &Path) -> Result<DetectedLayers>`.

Types:
- `DetectedLayers` — `layers: Vec<DetectedLayer>`, `stacks: Option<Vec<DetectedStack>>`
- `DetectedLayer` — `name: String`, `paths: Vec<String>`, `detected_by: Vec<DetectionLevel>`, `always_run: bool`, `depends_on: Vec<String>`
- `DetectionLevel` — enum: `Manifest`, `Directory`, `Framework`, `Config`, `ImportGraph`, `Override`
- `DetectedStack` — `name: String`, `root: String`, `layers: Vec<DetectedLayer>`, `detected_by: MonorepoTool`
- `MonorepoTool` — enum: `Nx`, `Turborepo`, `PnpmWorkspace`, `DirectoryConvention`
- `FrameworkPreset` — enum: `NextJs`, `Remix`, `SvelteKit`, `FastApi`, `Rails`, `Express`, `RustCli`

Detection levels:
1. **Manifest** — scan for `package.json` (parse `dependencies`/`devDependencies`), `Cargo.toml` (parse `[dependencies]`), `pyproject.toml`, `go.mod`, `Gemfile`. Extract framework/library signals.
2. **Directory** — static lookup table matching directory names to layer candidates. `src/server/`, `api/` → "backend"/"api". `terraform/`, `infra/` → "infrastructure". `deploy/`, `helm/`, `.github/workflows/` → "deployment". `pages/`, `app/`, `src/client/` → "frontend". `monitoring/`, `otel/`, `prometheus/` → "observability".
3. **Framework** — combine Level 1 deps + Level 2 dirs to apply `FrameworkPreset` rules. Level 3 promotes/reclassifies Level 2 candidates. Next.js (`next` in deps + `app/` or `pages/`) → frontend + api. Prisma (`prisma` or `@prisma/client`) → database. FastAPI/Django → api + backend.
4. **Config** — `Dockerfile`, `docker-compose.yml` → deployment. `vercel.json`, `netlify.toml` → deployment.
5. **Import graph** — **STUBBED**: return empty. Add `#[allow(dead_code)]` placeholder with `/// Phase 1 stub — real import graph analysis deferred` doc comment.
6. **Override** — if `.pice/layers.toml` exists at `project_root`, detection is SKIPPED ENTIRELY (PRDv2 line 756). Load and return the override config as `DetectedLayers`.

`always_run` inference: layer name contains "infrastructure", "deployment", or "observability" → `always_run = true`.

`depends_on` inference: from `FrameworkPreset` conventions only. Non-preset projects get empty `depends_on`.

Monorepo detection: if `nx.json`, `turbo.json`, or `pnpm-workspace.yaml` exists, parse the project graph (package list + roots) and produce `DetectedStack` entries. Each stack gets its own layer detection pass rooted at its directory. Fallback: multiple `package.json`/`Cargo.toml` files in subdirectories → `DirectoryConvention` stacks.

`to_layers_config(&self) -> LayersConfig` — convert detection output to a committable `LayersConfig` for `--write`.

Unit tests (minimum, using `tempfile::tempdir()`):
- Next.js fixture: `package.json` with `next` + `pages/` dir → detects frontend + api layers
- Prisma fixture: `schema.prisma` file → detects database layer
- Terraform fixture: `terraform/` dir → detects infrastructure layer with `always_run = true`
- Express fixture: `package.json` with `express` → detects api + backend
- FastAPI fixture: `pyproject.toml` with `fastapi` → detects api + backend
- Rails fixture: `Gemfile` with `rails` + `app/controllers/` → detects api + backend + database
- SvelteKit fixture: `package.json` with `@sveltejs/kit` → detects frontend + api
- Monorepo fixture: `nx.json` + two `packages/*/package.json` → detects 2 stacks
- Override fixture: `.pice/layers.toml` exists → detection skipped, override returned
- Config file fixture: `Dockerfile` + `.github/workflows/ci.yml` → deployment layer
- Multi-layer tagging: `pages/api/users.ts` matches both api and frontend globs

### Task 5: Create diff filter — `pice-core::layers::filter`

**File:** `crates/pice-core/src/layers/filter.rs`
**Action:** CREATE
**Description:** Implement git diff filtering by layer glob patterns. Pure text-processing function:

```rust
pub fn filter_diff_by_globs(full_diff: &str, globs: &[String]) -> String
```

Parse unified diff format:
- Split on `diff --git a/... b/...` headers
- Extract file path from each header (the `b/path` portion)
- Match file path against glob patterns using the `glob` crate's `Pattern::matches()`
- Reassemble only matching sections

Edge cases to handle:
- Binary files (`Binary files ... differ`) — match on the file path in the header
- Renames (`rename from ... rename to ...`) — match on the new path (`rename to`)
- New files (`--- /dev/null`) — match on the `+++ b/path`
- Deleted files (`+++ /dev/null`) — match on the `--- a/path`
- File paths with spaces — handle quoted paths in diff headers

Also implement:

## Notes

- **Level 5 (import graph) is stubbed** — returns empty results. This is intentional: Levels 1–4 cover the vast majority of real projects. Import graph analysis needs language-specific parsers and is a separate effort.
- **Seam checks are parsed but not executed** — the `[seams]` section in `layers.toml` is part of the schema and is parsed/validated, but seam check execution is Phase 3.
- **No parallel execution** — Phase 1 runs all layers and cohorts sequentially. Phase 5 adds worktree isolation and parallel cohort execution.
- **No adaptive pass allocation** — Phase 1 uses fixed pass count from tier. Phase 4 adds SPRT/ADTS/VEC.
- **Contract sourcing fallback** — if no per-layer contract exists, the plan-level contract is used for all layers. This is the v0.1 → v0.2 transition path.
- **glob crate** — added to `pice-core` only. The daemon doesn't need it directly — it calls `pice-core` functions that use glob internally.
- **Monorepo `[stacks]`** — parsed and stored but orchestrator-level multi-stack execution is a stretch goal. Phase 1 focuses on single-stack projects. Monorepo detection tests verify the schema round-trips correctly.

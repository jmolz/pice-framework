# Feature: PRDv2 Phase 4 — Adaptive Evaluation (Bayesian-SPRT, ADTS, VEC)

> Generated by `/plan-feature` — read CLAUDE.md and referenced files before implementing.

## Overview

Wire PRDv2 Feature 7 into the Stack Loops orchestrator. Introduces a new `pice-core::adaptive` module with three pure statistical algorithms (Bayesian-SPRT, Adversarial Divergence-Triggered Scaling, Verification Entropy Convergence), a halt-decision dispatcher, a Jacobson/Karels-style cost projector, and the per-pass loop in `stack_loops.rs` that replaces today's single-pass evaluation. Populates the already-scaffolded `LayerResult.{passes, halted_by, final_confidence, total_cost_usd}` fields from real provider observations (or mock observations for tests while Phase-1 provider wiring is still pending). Respects the ~96.6 % correlated-Condorcet ceiling as a hard invariant. Threads the workflow fields `min_confidence`, `max_passes`, `budget_usd` — parsed in Phase 2 but currently inert — into effective-per-layer resolution alongside `effective_tier_for`.

**Design commitments (locked after Q&A):**

- **ADTS uses three-level within-run escalation.** Level 1: extra pass with a fresh-context flag. Level 2: elevate the provider `effort` parameter (`high` → `xhigh` for Codex, equivalent for Claude) on the next pass. Level 3: halt with `adts_escalation_exhausted`. No mid-run tier re-issuance — tier is the committed contract depth decided at plan time.
- **Cost projection uses Jacobson/Karels smoothing.** `projected_next_cost = mean + 4·MAD` over observed per-pass costs (the same algorithm TCP uses for RTO in RFC 6298). Cold-start seed (pass 1): `budget_usd / max_passes`. Budget check is fail-closed — if the seed or the projection exceeds remaining budget, refuse to start the next pass.
- **`AdaptiveAlgo::None` is "no early-halt logic, budget still binding".** Budget cap and max_passes are universal guardrails; `None` only disables SPRT/ADTS/VEC halting. Users who want to bypass the budget explicitly raise `budget_usd` — no magical algorithm flags.

## User Story

As a developer running `pice evaluate` on a multi-layer change, I want evaluation to halt as soon as evidence is sufficient so that I do not pay for redundant passes and I see a clear halt reason (`sprt_confidence_reached`, `vec_entropy`, `budget`, `max_passes`, `adts_escalation_exhausted`) in the manifest — and I want confidence never to be claimed above the information-theoretic ceiling the dual-model architecture can actually deliver.

**Jacobson/Karels cost projector (from TCP RFC 6298 §3.4, stateless API):**

```rust
#[derive(Debug, Default, Clone, Copy)]
pub struct CostStats { pub n: u32, pub mean: f64, pub mad: f64 }

impl CostStats {
    pub fn new() -> Self { Self::default() }

    pub fn observe(&mut self, cost: f64) {
        self.n += 1;
        let n = self.n as f64;
        let new_mean = self.mean + (cost - self.mean) / n;
        let new_mad = self.mad + ((cost - new_mean).abs() - self.mad) / n;
        self.mean = new_mean;
        self.mad = new_mad;
    }

    pub fn project_next(&self, cold_start_seed: f64) -> f64 {
        if self.n == 0 { cold_start_seed } else { self.mean + 4.0 * self.mad }
    }
}
```

### Task 14: Write the adaptive pass loop with three-level ADTS

**File:** `crates/pice-daemon/src/orchestrator/adaptive_loop.rs`, `crates/pice-daemon/src/orchestrator/mod.rs`
**Action:** CREATE + MODIFY
**Description:** `async fn run_adaptive_passes(ctx) -> AdaptiveOutcome` where `AdaptiveOutcome = { passes: Vec<PassResult>, halted_by: Option<String>, final_confidence: Option<f64>, total_cost_usd: Option<f64>, escalation_events: Vec<EscalationEvent> }`. The last field is a per-pass audit trail of ADTS level changes (required by the determinism and audit-trail contract criteria).

Core pseudocode:
```
let mut cost_stats = CostStats::new();
let mut accum_cost = 0.0;
let mut next_pass_params = EvaluateParams { fresh_context: false, effort_override: None };
let mut adts_escalations_used: u32 = 0;

for pass_index in 1..=max_passes {
    // Pre-pass budget check
    let pre = decide_halt(algo, &passes, ..., accum_cost, &cost_stats, budget_usd);
    if pre.halt && pre.reason == Some(HaltReason::Budget) { halted_by = "budget"; break; }

    // Run provider(s)
    let (claude_pr, codex_pr_opt) = if algo == Adts {
        tokio::join!(run_provider("claude", pass_index, next_pass_params),
                     run_provider("codex", pass_index, next_pass_params))
    } else {
        (run_provider("claude", pass_index, next_pass_params).await, None)
    };
    passes.push(claude_pr);
    cost_stats.observe(claude_pr.cost_usd.unwrap_or(0.0));
    accum_cost += claude_pr.cost_usd.unwrap_or(0.0);
    if let Some(cr) = codex_pr_opt {
        passes.push(cr);
        cost_stats.observe(cr.cost_usd.unwrap_or(0.0));
        accum_cost += cr.cost_usd.unwrap_or(0.0);
    }
    insert_pass_events(conn, evaluation_id, pass_index, ...);   // write BEFORE halt decision

    // ADTS three-level escalation (only when algo == Adts)
    if algo == Adts {
        let verdict = run_adts(&paired_scores, adts_escalations_used, &adts_cfg)?;
        match verdict {
            AdtsVerdict::Continue => { next_pass_params = EvaluateParams::default(); }
            AdtsVerdict::ScheduleExtraPassFreshContext => {
                next_pass_params = EvaluateParams { fresh_context: true, effort_override: None };
                adts_escalations_used += 1;
                escalation_events.push(EscalationEvent::Level1 { at_pass: pass_index });
                continue;   // skip algorithm halt; force next pass
            }
            AdtsVerdict::ScheduleExtraPassElevatedEffort => {
                next_pass_params = EvaluateParams { fresh_context: true, effort_override: Some(Effort::XHigh) };
                adts_escalations_used += 1;
                escalation_events.push(EscalationEvent::Level2 { at_pass: pass_index });
                continue;
            }
            AdtsVerdict::EscalationExhausted => {
                halted_by = "adts_escalation_exhausted";
                escalation_events.push(EscalationEvent::Level3 { at_pass: pass_index });
                break;
            }
        }
    }

    // Post-pass halt decision (SPRT / VEC / None)
    let d = decide_halt(algo, &passes, ..., accum_cost, &cost_stats, budget_usd);
    if d.halt { halted_by = d.reason.unwrap().as_str().into(); final_confidence = Some(d.confidence); break; }
}
if halted_by.is_none() { halted_by = "max_passes"; }
total_cost_usd = Some(accum_cost);
```

**Critical invariants:**
- **Per-pass metrics are written BEFORE the halt decision check.** This guarantees budget-halted layers still have the triggering-pass cost persisted.
- **`insert_pass_events` uses the transaction pattern from `metrics/store.rs`** — single COMMIT at end of loop would lose partial data on crash.
- **`next_pass_params` is reset to `default()` after a non-ADTS pass** — ADTS flags only apply to the immediately following pass, not permanently.
- **`escalation_events` is a new field** added to `LayerResult` in Task 14b (below) as a forward-compatible `Option<Vec<EscalationEvent>>` with `#[serde(default, skip_serializing_if = "Option::is_none")]`.

Exports `run_adaptive_passes`, `EscalationEvent`, `AdaptiveOutcome`.

## Notes

### Design Decisions (locked after Q&A)

- **Algorithm selection is per-phase, user picks ONE via `evaluate.adaptive_algorithm`.** SPRT is default (statistical halting); ADTS is for teams that prioritize cross-model disagreement detection; VEC is for teams that want entropy-based convergence; `None` disables early halting. This matches PRDv2:1146 literally.
- **ADTS uses three-level within-run escalation** (borrowed from MCTS / adaptive MCMC): Level 1 (first divergence) → extra pass with `fresh_context=true`; Level 2 (still divergent) → extra pass with `effort=xhigh`; Level 3 (still divergent at `max_divergence_escalations`) → halt with `adts_escalation_exhausted`. Mid-run formal tier re-issuance is explicitly rejected — tier is the committed contract depth decided at plan time, and changing it would invalidate the evaluator's cached work and muddle the audit trail. The three-level escalation captures the PRDv2:1127 intent ("escalate to next tier") by escalating *compute*, not *identity*.
- **Budget projection uses Jacobson/Karels smoothing** (`mean + 4·MAD`, same algorithm TCP uses for RTO in RFC 6298 §3.4). Properties that beat the alternatives: fail-closed (over-estimates when cost variance is high), self-tuning (no magic constants per workload), O(1) state. Cold-start seed (pass 1) uses `budget_usd / max_passes` — if that exceeds remaining budget, refuse to start. Max-of-observed and EWMA were considered but neither has the adaptive-variance property Jacobson/Karels gets for free.
- **`AdaptiveAlgo::None` respects the budget and max_passes guardrails.** Budget is a financial safety rail, not a strategy choice. Users who want to run past budget raise `budget_usd` explicitly — no magical algorithm flags that bypass safety. This principle matches PRDv2's "Fail closed on evaluation" rule and is stricter than one might naively design. Distinct from omitting the field (which uses the default `BayesianSprt`).
- **Phase-1 provider wiring is NOT a prerequisite for Phase 4.** The adaptive algorithms are pure functions over `PassObservation` streams. The stub provider supplies deterministic observations for integration tests. When real providers land (deferred Phase 1 carry-over), the adaptive loop Just Works because it takes `PassResult` by value, not by provider identity.
- **Beta posterior math is hand-rolled.** Adding `statrs` would bloat pice-core and break the "zero external math deps" posture. The digamma asymptotic series is well-documented (Abramowitz & Stegun 6.3) and the recurrence handles small arguments. The accuracy target (1e-6) is achievable with the standard 6-term asymptotic + the recurrence.
- **The `HaltReason` enum is locked to its serde tag via `as_str()`** — mirror the `ExitJsonStatus` pattern. The unit test `halt_reason_as_str_matches_serde_snake_case` guards drift. Same pattern applies to `EscalationEvent`.
- **Determinism is a first-class contract criterion (threshold 10).** Any non-deterministic source — hashmap iteration order, unsynchronized parallel-tokio task ordering for ADTS `tokio::join!`, timestamp inclusion in non-timestamp fields — breaks reproducibility and must be purged. The `determinism_across_two_identical_runs` test locks this.
- **Cost reconciliation is enforced by post-hoc SQL**, not by pre-write validation. Writes go to both the manifest file and `pass_events` SQLite table independently; reconciliation verifies they agree. Any divergence is a write-path bug.

### Trade-offs Accepted

- **Confidence reporting is posterior-mean, not credible-interval.** A credible-interval approach (e.g. lower bound of 95% CI) would be more conservative but harder to explain in UI. The posterior-mean-capped-at-ceiling approach is consistent with PRDv2:1079–1090's tabular presentation and is sufficient for the halting decision (which uses log-likelihood-ratio, not the reported confidence).
- **ADTS only supports pairs (Claude + Codex).** A future generalization to N-model divergence (ρ-matrix) could be added, but v0.4 matches the dual-model architecture of the current stack.
- **Jacobson/Karels projection uses an arithmetic-mean MAD update rather than a full RFC 6298 EWMA (α=1/8, β=1/4).** The arithmetic mean is simpler to reason about, fully deterministic (no smoothing-factor sensitivity), and sufficient for the bounded per-layer horizon (`max_passes` is typically 5–10, not thousands). An EWMA variant would be a drop-in replacement if a future phase needs reaction-speed tuning.
- **ADTS escalation uses `effort=xhigh` unconditionally at Level 2.** A provider without an `effort` dimension (e.g. a local model without sampling-temperature control) will see this as a no-op. Level 2 still fires `fresh_context=true`, so some forward progress is made. A provider-capability query to skip Level 2 when effort is unsupported is a Phase 5 refinement.
- **VEC entropy uses Beta posterior only.** If a future phase introduces non-Beta posteriors (e.g. Dirichlet for multi-outcome), VEC will need a generalization. For v0.4, Beta is sufficient for binary success/failure observations.
- **`AdaptiveAlgo::None` denies the escape hatch some users might expect.** A user who truly wants unbounded evaluation must raise `budget_usd` explicitly. This is the conservative default; making bypass automatic would create a foot-gun.

### Known Limitations

- The correlated-Condorcet prediction is itself an approximation — real LLM correlations fluctuate by task domain. The ±0.02 calibration tolerance accommodates this but would tighten as more empirical data accumulates.
- Cost measurement relies on provider-reported `costUsd`. **Pass-6 Codex Critical #1 closed** via a capability gate (`ProviderCapabilities.costTelemetry`): a provider that does not declare `costTelemetry: true` cannot run adaptive evaluation with `budget_usd > 0`. The daemon fails closed at layer start rather than running the loop on synthetic `budget_usd / max_passes` seed debits. Teams can use such providers with `budget_usd = 0` (no financial enforcement) or must wire real per-pass `costUsd` on `evaluate/create`. The shipped stub and provider-claude-code (once cost telemetry lands) qualify; provider-codex and any third-party provider must opt in explicitly.
- The adaptive loop is sequential within a layer but parallel across layers (existing DAG parallelism). Future work could add mid-layer parallelism (e.g. two SPRT streams with different priors) but v0.4 keeps the per-layer loop serial for determinism.

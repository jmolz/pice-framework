## Notes

**GitHub SVG dark mode:** GitHub's markdown renderer strips `<style>` tags and `@media` queries from inline SVGs. The only reliable approach is the `<picture>` element with `<source media="(prefers-color-scheme: dark)">`. This requires two SVG files per diagram but is battle-tested across major open-source projects.

**VHS mock approach:** The .tape file runs a mock shell script rather than the real `pice evaluate` command because the real command requires `ANTHROPIC_API_KEY` and `OPENAI_API_KEY`. The mock echoes the exact output from `output.rs` with timing delays for a natural streaming feel. This makes the GIF reproducible in CI or by contributors without API credentials.

**SVG vs Excalidraw export:** We create hand-crafted SVGs rather than exporting from Excalidraw because: (1) GitHub's SVG sanitizer is strict ‚Äî hand-crafted SVGs can avoid problematic elements, (2) we need two color variants, and (3) self-contained system fonts are easier to guarantee. The `.excalidraw` files serve as the editable design source for future iteration.

**Known output.rs bugs:** Lines 55 and 71 in `output.rs` are missing closing `‚ïë` characters on adversarial challenge lines and the summary line. The GIF mock script should match the README example (which has them) rather than the buggy source output, since the README is the public-facing representation. These bugs should be fixed separately.
